About Project

Fashion recommendation systems powered by AI have transformed the way consumers discover clothing and accessories. However, these systems often lack transparency, leaving users in the dark about why certain recommendations are made. We begin by establishing the fundamentals of AI-driven fashion recommendations and the challenges they face, such as subjective fashion preferences and the need to balance personalization and diversity. This explores evaluation metrics for measuring the effectiveness of explainability, considering user satisfaction, trust, and system performance. Ethical concerns related to bias and fairness in fashion recommendations are discussed, with explainability playing a crucial role in addressing these issues.

Technologies

Technologies Used:

Before we dive into the setup process, let's take a quick look at the cutting-edge technologies, algorithms, and other tools powering your FRS project:


👉 Backend Technologies:

  • Python: The backend of your FRS project is developed using Python, a versatile and powerful programming language known for its simplicity and readability.
  • Django: Django, a high-level Python web framework, is utilized for building robust and scalable web applications, providing a clean and pragmatic design.
  • Django REST Framework (DRF): DRF is employed for creating RESTful APIs in your Django application, facilitating seamless communication between the frontend and backend.
  • PostgreSQL: PostgreSQL, a powerful open-source relational database management system, is chosen for storing and managing your FRS data securely.
  • TensorFlow: TensorFlow is a popular open-source machine learning framework developed by Google. It provides a comprehensive ecosystem of tools, libraries, and community resources for building and deploying machine learning models.
  • ResNet50: ResNet50 is a pre-trained convolutional neural network architecture commonly used for image classification tasks. It is part of the ResNet (Residual Neural Network) family and is known for its deep architecture and high accuracy in image recognition tasks.
  • NumPy: NumPy is a fundamental package for scientific computing with Python. It provides support for large, multi-dimensional arrays and matrices, along with a collection of mathematical functions to operate on these arrays efficiently.
  • os: The os module in Python provides a portable way of using operating system-dependent functionality. It is used in the script for interacting with the file system to handle file paths and directory operations.
  • tqdm: tqdm is a fast, extensible progress bar library for Python. It provides a simple and intuitive way to display progress bars for iterative processes, making it easier to track the progress of tasks such as image processing.
  • Pickle: Pickle is a Python module used for serializing and deserializing Python objects. It allows objects to be saved to a file and loaded back into memory later, which is useful for storing and retrieving complex data structures such as feature vectors and filenames.

👉 Frontend Technologies:

  • JavaScript: The frontend of your FRS project is developed using JavaScript, the cornerstone of web development, enabling dynamic and interactive user interfaces.
  • React Native: React Native, a popular JavaScript framework, is employed for building cross-platform mobile applications with native-like performance, ensuring a smooth and consistent user experience across different devices.
  • Expo: Expo is utilized as a set of tools, libraries, and services for developing React Native applications more efficiently, allowing for rapid prototyping and seamless deployment.

👉 Algorithms Used:

  • ResNet50: ResNet50, a deep convolutional neural network architecture, is employed for image processing tasks in your FRS project. Its ability to extract features from images makes it suitable for analyzing fashion images and facilitating recommendations based on visual data.
  • K-Nearest Neighbors (KNN): KNN algorithm is utilized for recommendation purposes in your FRS project. KNN helps recommend relevant content or actions to users based on their preferences and past interactions.
  • LIME (Local Interpretable Model-agnostic Explanations): LIME is employed for explainable artificial intelligence (xAI) in your FRS project. It provides insights into how machine learning models make decisions, allowing for transparency and interpretability, especially crucial in healthcare applications where understanding model predictions is essential.

👉 Other Tools and Techniques:

  • Web Scraping: Utilized for gathering data like product images, links etc. Web scraping enables your FRS system to enrich its knowledge base and provide more comprehensive insights to users.
  • Image-to-Base64 Conversion: Images are converted to base64 format for efficient storage and transmission within the application. This technique enhances performance and reduces the overhead of handling image data, ensuring seamless integration with backend services.
  • Additional APIs for Enhanced Explanations: Integration with external APIs, such as medical dictionaries, terminology databases, or symptom checkers, enhances the explanatory capabilities of your FRS system. These APIs provide additional context and definitions to users, improving their understanding of medical terms and conditions.

Project Setup Guide

Welcome to the setup guide for developing a Fashion Flairhub application!
Follow these steps to set up your development environment and start working on the project.

Prerequisites Before getting started, ensure you have the following prerequisites installed on your Ubuntu system:

  • 1. Create Project Directory: Begin by creating a directory for your FRS project. You can name it whatever you like. For example: mkdir FRS
    cd FRS
  • 2. Clone Frontend and Backend Repositories: git clone
  • 3. Set Up Virtual Environment: git clone
  • 4. Navigate to the project directory: cd project_dir
  • 5. Create a virtual environment named "dev": python3 -m venv dev
  • 6. Activate the virtual environment: source dev/bin/activate
  • 7. Install backend dependencies: pip install -r requirements/development.txt
  • 8. Apply database migrations: python manage.py migrate
  • 9. Start the backend development server: python manage.py runserver 0.0.0.0:8000
  • 10. Navigate to the FRS frontend directory: cd ../frs-frontend
  • 11. Install dependencies and Start the frontend development server: npm install
    npm start

You have successfully set up your development environment for the FRS application. Happy coding!
